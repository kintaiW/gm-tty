⚡

单二进制部署

前端资源编译期内嵌到可执行文件，Server 仅 4.6 MB，Agent 仅 2.9 MB，无运行时依赖。

🔀

反向隧道穿透

Agent 主动连接公网 Server，无需公网 IP、端口映射或 P2P 打洞，防火墙友好。

🔒

Token 会话隔离

每个 Token 绑定独立会话，CSPRNG 生成 64 字符 hex，一时刻仅允许一个 Agent + 一个 Web 连接。

🔄

指数退避重连

Agent 网络中断后自动重连，初始 1s，最大 60s，响应 SIGINT/SIGTERM 优雅退出。

🖥️

全功能终端

基于 xterm.js，支持 256 色、窗口 Resize、滚动缓冲，原生 PTY 无缝透传。

🦀

纯 Rust 实现

tokio 异步运行时，零拷贝转发，内存安全，跨平台支持 Linux / macOS / Windows。

工作原理

三个组件，两条 WebSocket 连接，一次数据转发

1
Agent 注册

Agent 主动反向连接 Server 的 /ws/agent/{token}，Server 验证 Token 并创建待桥接会话。

2
用户访问

浏览器加载内嵌前端页面，xterm.js 连接 /ws/web/{token}，Server 匹配 Token 完成桥接。

3
透明转发

Server 以 tokio::mpsc channel 桥接两端，键盘输入 → PTY stdin，PTY stdout → 终端渲染，零解析开销。

快速开始

三步完成内网 Shell 穿透

1 编译 
# 需要 Rust 1.75+
git clone https://github.com/GateMate-TQH/gm-tty
cd gm-tty
cargo build --release

# 产出
target/release/gm-tty-server   # 4.6 MB，含前端
target/release/gm-tty-agent    # 2.9 MB
2 公网 Server 启动 
# 方式 A：自动生成临时 Token（开发测试）
./gm-tty-server

# 方式 B：配置文件
cat > server.toml <<'EOF'
listen_addr = "0.0.0.0:7093"
tokens = ["your-secret-token"]
EOF
./gm-tty-server server.toml

# 生成随机 Token
./gm-tty-server gen-token
3 内网 Agent 连接 
# 支持 ws:// (明文) 和 wss:// (TLS/反向代理)
GM_TTY_SERVER=wss://your-server.example.com \
GM_TTY_TOKEN=your-secret-token \
./gm-tty-agent

# 浏览器访问
https://your-server.example.com/terminal/your-secret-token
